# Campaign Engine Input

Fill this out and hand it to the skill to start a run. Only "Event" is required — everything else has a sensible default.

## Event
[Name of the event, or a short description if it doesn't have a common name yet]

## Sources (optional)
[Any links, articles, or notes you already have. The engine will still run its own research pass regardless — this just seeds it.]

## Status
- [ ] Active / ongoing
- [ ] Resolved / historical

If left blank, the engine determines this during research and states it explicitly in the frame post/section.

## Output Format
- [ ] X thread (default)
- [ ] Newsletter
- [ ] Other (name it — the engine will tell you if a format spec doesn't exist yet for it)

## POV Override (optional)
[Leave blank to let the engine select the POV using the selection criteria in SKILL.md: maximum uncertainty zone, physical access, exposure to escalation beats, natural information gating.]

## Constraints (optional)
[Anything the engine should specifically avoid or include. For example: "no casualty detail," "keep the auditor's identity anonymous," "focus only on the first 48 hours."]
